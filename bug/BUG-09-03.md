# BUG-09-03: 부분 일치 답변이 오답으로 판정됨

## 현상
- 퀴즈 정답이 "미국 애틀랜타"일 때 사용자가 "애틀랜타"라고 답하면 오답 처리
- 퀴즈 정답이 "코카콜라 클래식"일 때 "코카콜라"라고 답하면 오답 처리
- 반대로 정답이 "서울"일 때 "대한민국 서울특별시"라고 답하면 오답 처리 (정답이 답변에 포함된 경우도 미처리)

## 근본 원인
`services/quiz/judge.ts`의 `judgeAnswer()`가 정규화 후 완전 일치만 검사:
```typescript
export function judgeAnswer(userAnswer: string, correctAnswer: string): boolean {
  if (!userAnswer || !correctAnswer) return false;
  return normalizeAnswer(userAnswer) === normalizeAnswer(correctAnswer);
  // 포함 관계 미확인
}
```

## 수정 방향
정규화 후 완전 일치 → 추가로 포함 관계 확인:
1. `normalUser === normalCorrect` (완전 일치)
2. `normalCorrect.includes(normalUser)` (사용자 답이 정답에 포함 — 핵심어 입력)
3. `normalUser.includes(normalCorrect)` (정답이 사용자 답에 포함 — 더 구체적으로 입력)
- 오남용 방지: 최소 2글자 이상인 경우에만 포함 관계 체크 적용
