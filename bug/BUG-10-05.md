# BUG-10-05: 퀴즈 step 수정 시 중복 단계 충돌 미처리

## 현상
- 퀴즈 편집 시 step 변경 UI 자체가 없음
- 만약 같은 step을 지정하면 DB UNIQUE(project_id, step) 제약으로 오류 발생

## 수정 방향
- QuizEditor 편집 행에 step 드롭다운 추가
- PATCH `/quizzes/[quizId]` API: step 변경 시 충돌 단계와 이후 단계를 순차적으로 1씩 뒤로 밀기

## 알고리즘
- 새 step < 기존 step (앞으로 이동): [newStep, oldStep-1] 범위 퀴즈 step+1
  - 처리 순서: step 내림차순 (conflict 방지)
- 새 step > 기존 step (뒤로 이동): [oldStep+1, newStep] 범위 퀴즈 step-1
  - 처리 순서: step 오름차순 (conflict 방지)
- UNIQUE 제약 회피: 대상 퀴즈를 먼저 임시 step(99)으로 이동 후 shift, 최종 이동
