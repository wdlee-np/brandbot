# BUG-11-02: 브랜드봇 재활성화 시 Gemini 컨텍스트 캐시 미갱신

## 현상
비활성화된 브랜드봇을 다시 활성화해도 이전에 생성된 Gemini context cache가 그대로 유지됨.
브랜드 정보(brand_info_path)나 퀴즈 내용이 변경된 후 재활성화해도 봇이 이전 정보 기반으로 응답하거나 퀴즈를 잘못 냄.

## 원인
`PATCH /api/admin/projects/[id]`에서 `status → active` 전환 시 `context_cache_id`를 초기화하지 않아 기존 캐시를 계속 사용함.

## 수정 내용
`app/api/admin/projects/[id]/route.ts`에서 `status === 'active'`로 활성화할 때 `context_cache_id: null`을 함께 업데이트하여 다음 요청 시 캐시가 새로 생성되도록 강제.

## 수정 파일
- `app/api/admin/projects/[id]/route.ts`
