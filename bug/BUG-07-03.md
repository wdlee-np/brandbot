# BUG-07-03: 퀴즈 완료 상태 로컬 스토리지 미저장 — 쿠폰 지급 신뢰성 보완

## 현상
- 서버 `participants.quiz_progress`는 정상 저장되나, 클라이언트 상태(`quizProgress`)는
  페이지 새로고침 시 서버에서 다시 받아오기 전까지 0으로 표시됨
- 새로고침 직후 쿠폰 지급 조건(`quizProgress >= 3`)을 클라이언트가 인식 못해
  reward 페이지로 자동 이동이 지연되거나 누락될 수 있음
- 서로 다른 3개의 퀴즈(step 1, 2, 3)를 각각 풀었는지 여부를 클라이언트가 독립적으로
  추적하지 않아 대화 흐름과 퀴즈 완료 흐름이 분리되지 않음

## 수정 방향
- 퀴즈 단계별 정답 시 `localStorage`에 `bb_quiz_done_{projectId}_{tokenSuffix}` 키로
  `{ completedSteps: [1, 2, 3] }` 형태로 저장
- `useChat` 초기화 시 localStorage 값과 `initialQuizProgress`(서버값) 중 큰 값 사용
- 3개 완료 감지 시 쿠폰 API 자동 호출 → 쿠폰 정보를 reward 페이지에 전달
- localStorage가 없더라도 서버 `quiz_progress` 기반으로 폴백 동작 보장
