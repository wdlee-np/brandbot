# BUG-08-06: 퀴즈 pass 기록이 대화 이력과 혼용되어 관리됨

## 증상
퀴즈 통과 상태가 localStorage의 `bb_quiz_done_*` 키에 저장되지만, 대화 이력 초기화 시 함께 초기화될 위험이 있음.
오답 횟수(`wrongAttempts`)는 아예 저장되지 않아 페이지 새로고침 시 초기화됨.

## 원인
`hooks/useChat.ts`에서:
- `bb_quiz_done_*`: 완료 단계 배열 저장 (대화 이력과 독립적이나 구조가 분산됨)
- `bb_quiz_asked_*`: 출제 단계 배열 저장 (중복 출제 방지용)
- 오답 횟수를 저장하는 키가 없음
- 퀴즈 상태가 여러 키에 분산되어 관리가 복잡함

## 수정 방법
`quizSessionState`로 통합:
```typescript
interface QuizStepState { passed: boolean; wrongAttempts: number; }
type QuizSessionState = { 1: QuizStepState; 2: QuizStepState; 3: QuizStepState };
// localStorage 키: bb_quiz_state_{projectId}_{token.slice(-10)}
```
- 대화 이력(`bb_history_*`)과 완전히 분리된 독립 키 사용
- 기존 `bb_quiz_asked_*`, `bb_quiz_done_*` 키 제거
- 3단계 모두 `passed = true`이면 쿠폰 지급 트리거

## 영향 범위
- `hooks/useChat.ts`
