# BUG-08-04: 퀴즈 오답 처리 로직 없음 (힌트 2회 후 정답 공개)

## 증상
현재 퀴즈 오답 시 아무런 판정 없이 일반 대화처럼 흘러감.
오답 횟수를 추적하지 않으므로 힌트 제공 로직과 정답 공개 로직이 전혀 동작하지 않음.

## 원인
1. `app/api/chat/[projectId]/message/route.ts`에서 사용자 답변을 판정하는 로직이 없음.
   별도의 `/quiz/answer` 엔드포인트를 호출하는 구조였지만 이는 메시지 응답 후 비동기로 호출되어 타이밍 이슈 발생.
2. `hooks/useChat.ts`에서 오답 횟수(`wrongAttempts`)를 추적하지 않음.

## 수정 방법
1. `types/index.ts`에 `ChatApiResponse`에 `quizResult` 필드 추가
2. `message/route.ts`에서 `quizAnswerContext` 수신 시 서버에서 직접 정답 판정:
   - 정답: DB `quiz_progress` 업데이트, `quizResult.passed = true`
   - 1회 오답: 힌트1 지시어 생성, `quizResult.passed = false`
   - 2회 오답: 힌트2 지시어 생성, `quizResult.passed = false`
   - 3회 오답: DB force-pass, 정답 공개 지시어, `quizResult.passed = true, revealed = true`
3. `hooks/useChat.ts`에서 `quizSessionState`로 오답 횟수 관리

## 영향 범위
- `types/index.ts`
- `app/api/chat/[projectId]/message/route.ts`
- `hooks/useChat.ts`
