# BUG-07-01: Gemini API 대화 기억 없음 — 로컬 스토리지 히스토리 유지

## 현상
- `useChat` 훅의 `historyRef`가 in-memory이므로 페이지 새로고침 시 대화 이력이 초기화됨
- Gemini API 자체는 stateless이므로 클라이언트가 이전 대화를 전송해야 문맥을 유지함
- 새로고침 후 히스토리가 없으므로 Gemini가 이전 대화를 전혀 알지 못함
- 퀴즈가 이미 출제된 이후에도 매 메시지마다 동일한 퀴즈 문제를 context에 주입해
  Gemini가 같은 퀴즈를 반복해서 출제함

## 원인
1. `historyRef.current`는 React ref로 페이지 언마운트 시 소멸
2. `/api/chat/[projectId]/message` 라우트가 매 요청마다 `nextStep` 퀴즈 문제를 무조건 주입
3. 클라이언트가 "이미 이번 세션에서 출제된 퀴즈 단계"를 서버에 전달하지 않음

## 수정 방향
- `useChat` 훅 초기화 시 `localStorage`에서 대화 이력 로드 (최대 40개 메시지 = 20턴)
- 메시지 교환마다 `localStorage` 업데이트
- `askedQuizSteps: number[]` 를 API 요청 body에 포함하여 이미 출제된 단계는 context에서 제외
- `localStorage` 키: `bb_history_{projectId}_{tokenSuffix}`, `bb_quiz_asked_{projectId}_{tokenSuffix}`

## 영향 파일
- `hooks/useChat.ts`
- `app/api/chat/[projectId]/message/route.ts`
