# BUG-11-01: 퀴즈 번호 변경 시 밀기 대신 교체(스왑) 방식 적용

## 현상
어드민에서 퀴즈 step을 변경할 때 충돌하는 번호가 있으면 해당 범위의 모든 step을 한 칸씩 밀어낸다.
예: Q1→Q3 변경 시 Q2, Q3가 각각 Q3, Q4로 밀림 → 의도치 않은 순서 변경 발생.

## 원인
`shiftQuizSteps` 함수가 변경 전·후 범위 내 모든 퀴즈를 순차 이동시키는 방식이었음.

## 수정 내용
`app/api/admin/projects/[id]/quizzes/[quizId]/route.ts`의 `shiftQuizSteps`를 `swapOrMoveQuizStep`으로 교체.
- 목표 step에 충돌하는 퀴즈가 있으면 두 퀴즈의 step을 **교체(swap)**
- 충돌이 없으면 단순 이동

## 수정 파일
- `app/api/admin/projects/[id]/quizzes/[quizId]/route.ts`
