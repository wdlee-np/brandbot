# BUG-10-04: 퀴즈 활성화 조건 3개 고정 + Q번호 최대 Q3까지만 선택 가능

## 현상
1. 퀴즈가 1~2개만 있어도 활성화 불가 (3개 미만이면 비활성)
2. 수동 퀴즈 추가 시 Q1~Q3까지만 선택 가능 (실제로는 최대 5개 허용 필요)

## 근본 원인
- `ProjectDetailClient.tsx`: `disabled={currentQuizzes.length < 3}` 하드코딩
- `QuizEditor.tsx`: `canActivate = quizzes.length >= 3`, step 선택 `[1,2,3]` 고정
- `app/api/admin/projects/[id]/route.ts`: 활성화 시 `count < 3` 체크
- `app/api/admin/projects/[id]/quizzes/route.ts`: `step IN [1,2,3]` 검증
- `supabase/migrations/001_initial_schema.sql`: `step IN (1,2,3)` CHECK constraint
- `participants.quiz_progress`: `BETWEEN 0 AND 3` 제약

## 수정
- 활성화 조건: 1개 이상이면 활성화 가능
- 퀴즈 step 범위: 1~5 (DB CHECK constraint 제거, 앱 레벨 검증)
- 채팅 흐름: 실제 프로젝트 퀴즈 목록 기반으로 nextStep 계산 (동적)
- quiz_progress: BETWEEN 0 AND 5로 확장
