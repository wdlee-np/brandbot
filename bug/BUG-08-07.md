# BUG-08-07: 퀴즈 요청 시 이미 통과한 단계를 다시 출제하거나 시스템 설명 거부

## 증상
1. 사용자가 "퀴즈 풀고 싶어요"라고 하면 이미 통과한 단계의 퀴즈를 다시 출제하는 경우 발생
2. 사용자가 "쿠폰은 어떻게 받나요?", "퀴즈가 몇 개 있어요?" 등 시스템 관련 질문을 하면 AI가 응답을 거부하거나 무관한 답변을 함

## 원인
1. `message/route.ts`에서 `nextStep = quizProgress + 1`로 다음 퀴즈를 결정하지만,
   클라이언트의 localStorage 통과 기록(`quizSessionState`)과 서버 DB의 `quiz_progress`가 동기화되지 않을 수 있음.
   `passedQuizSteps`를 서버에 전송하지 않아 서버가 클라이언트 상태를 모름.
2. `brand-system.txt`에 퀴즈/쿠폰 시스템 설명 허용 규칙이 없어 AI가 관련 질문을 거부함.

## 수정 방법
1. `message/route.ts`에서 `passedQuizSteps` 수신:
   - DB `quiz_progress`와 `passedQuizSteps.length` 중 큰 값을 `effectiveProgress`로 사용
   - `nextStep = effectiveProgress + 1`
2. `hooks/useChat.ts`에서 `sendMessage` 시 `passedQuizSteps` 전송
3. `brand-system.txt`에 "퀴즈와 쿠폰 시스템에 대한 질문에는 친절하게 설명 가능" 규칙 추가

## 영향 범위
- `app/api/chat/[projectId]/message/route.ts`
- `hooks/useChat.ts`
- `ai/prompts/brand-system.txt`
