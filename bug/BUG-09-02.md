# BUG-09-02: 이미 통과한 퀴즈 단계에서 계속 힌트 제공

## 현상
- 퀴즈 1단계를 이미 통과했음에도 다시 1단계 퀴즈 힌트를 제공
- 예: "아직 통과 못했어요... 힌트: [1단계 퀴즈 힌트]" (이미 1단계 통과 후 2단계 진행 중인 상황)
- 서버가 `nextStep`을 `passedQuizSteps.length + 1`의 단순 산술로 계산하여 실제 통과하지 않은 최초 단계를 찾지 못함

## 근본 원인
`app/api/chat/[projectId]/message/route.ts`:
```typescript
// 현재: 단순 산술 (passedQuizSteps = [1,3]이면 nextStep = 3, 실제론 2가 미통과)
const localProgress = passedQuizSteps.length;
const effectiveProgress = Math.max(dbQuizProgress, localProgress);
const nextStep = effectiveProgress + 1;
```
- `passedQuizSteps`가 집합(Set)이므로, 연속되지 않을 수 있음 (예: [1,3])
- 올바른 로직: 1→2→3 순서로 순회하며 통과하지 않은 첫 번째 단계 탐색

## 수정 방향
```typescript
const allPassedStepsSet = new Set([
  ...Array.from({ length: dbQuizProgress }, (_, i) => i + 1),
  ...passedQuizSteps,
]);
const nextStep: number | null = ([1, 2, 3] as const).find(s => !allPassedStepsSet.has(s)) ?? null;
```
- `nextStep === null`이면 모든 단계 완료, 퀴즈 미출제

## 영향 파일
- `app/api/chat/[projectId]/message/route.ts`

## 참고
- BUG-09-03(정답 미판정)이 근본 원인이었던 케이스가 많음: 퀴즈가 통과되지 않아 계속 같은 단계에서 힌트 제공
- 두 버그 모두 수정해야 완전히 해소됨
