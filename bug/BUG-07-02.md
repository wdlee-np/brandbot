# BUG-07-02: Gemini가 임의 퀴즈 생성 — DB 퀴즈 데이터 강제 사용

## 현상
- Gemini가 system prompt의 지시를 무시하고 자체적으로 퀴즈 문제를 만들어 출제하는 경우 발생
- 실제 DB에 저장된 퀴즈(`quizzes` 테이블)와 다른 내용이 사용자에게 보임
- 정답 판정은 DB 기준이므로 Gemini가 다른 문제를 내면 정답을 맞혀도 오답 처리됨

## 원인
- API 응답에 `quizQuestion` (DB 원본)이 이미 포함되어 있으나 클라이언트가 이를 표시하지 않음
- `ChatMessage` 컴포넌트는 Gemini 응답 텍스트만 렌더링하여 DB 문제와 불일치 가능
- 시스템 프롬프트의 퀴즈 출제 지시만으로는 LLM 준수를 보장할 수 없음

## 수정 방향
- `isQuizMode`가 true이고 `quizQuestion`(DB 원본)이 있을 때,
  ChatWindow에서 DB 원본 퀴즈 문제를 공식 질문 카드로 별도 표시
- Gemini 응답 텍스트는 유도 멘트로 활용, 실제 문제는 DB에서 가져온 것을 우선 표시
- system prompt에 "퀴즈 문제 텍스트를 임의로 변경하지 말 것" 지시 강화
